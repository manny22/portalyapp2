Revisa todo el proyecto generado para Janora como si fueras un Senior Software Architect.

Valida específicamente:

1. Que el monorepo funcione correctamente con pnpm.
2. Que apps/web y apps/api estén bien separados.
3. Que el backend aplique arquitectura limpia de forma pragmática.
4. Que no exista lógica de negocio en controllers.
5. Que no exista lógica de negocio compleja dentro de componentes React.
6. Que las rutas protegidas del frontend respeten los roles.
7. Que el backend aplique RBAC correctamente.
8. Que exista tenant isolation por propertyId.
9. Que ningún usuario pueda consultar datos de otra copropiedad.
10. Que las validaciones de DTOs sean suficientes.
11. Que Prisma tenga relaciones, índices y restricciones correctas.
12. Que el registro de ingreso y salida no permita inconsistencias.
13. Que el login, refresh token y logout estén correctamente implementados.
14. Que el frontend tenga estados de loading, error y empty state.
15. Que el README permita ejecutar el proyecto desde cero.

Corrige todos los problemas encontrados y entrega una versión mejorada del código.
